# 自律施工・半自律化の技術概念

建設機械の自律施工に必要なソフトウェア技術・アーキテクチャ・安全要件を、自動車の自動運転との比較を交えて整理する。対象読者は技術担当者・アーキテクト。

---

## 1. 自律施工とは

### 自律化の4レベル定義

建設機械の自律化は業界横断の統一規格が存在しないため、以下のように独自整理する（自動車のSAEレベルを参考に実態に即した定義）。

| レベル | 名称 | 概要 | 操作員の役割 |
|--------|------|------|--------------|
| L0 | 施工ガイダンス | センサ・GNSSにより設計値との差分をHMI表示。判断・操作は人間 | 全操作を実施。ガイダンス情報を参照 |
| L1 | 半自律（軸補助） | 特定の軸（バケット角度・高さ等）を自動制御。他は手動 | 主操作を実施。補助軸は自動 |
| L2 | 条件付き自律 | 定義済み作業サイクル（掘削・旋回・放土）を自動化。異常時に人間が介入 | 監視・異常対応 |
| L3 | 高度自律 | 施工計画に基づき一連の作業を自律実行。複雑地形でも継続 | 遠隔監視のみ |
| L4 | 完全自律 | 施工開始から完了まで人間の介入なし | 不要（緊急停止のみ保持） |

遠隔操作（テレオペレーション）は上記とは別カテゴリ。通信遅延と人的負荷の課題はあるが、危険環境での即時導入手段として有効。

### 自律施工が求められる背景

- **人手不足**：建設技能労働者の高齢化・減少。熟練オペレーター技能の継承困難
- **危険環境**：急傾斜地・土石流リスク・爆発物処理現場での無人化需要
- **夜間・連続施工**：工期短縮のための24時間稼働
- **精度向上**：設計値通りの出来形管理。ICT施工（i-Construction等）の推進

## 2. 自動車の自動運転との比較

### SAEレベルとの対応（参考）

| SAE（自動車） | 建設機械類比 | 主な差異 |
|---------------|--------------|----------|
| L0：手動運転 | L0：施工ガイダンス | ガイダンスHMIの有無のみ |
| L1/L2：運転支援 | L1：軸補助制御 | 作業機制御が3次元。車は2次元の操舵 |
| L3：条件付き自動運転 | L2：作業サイクル自動化 | 場面切替の判断が施工内容依存で複雑 |
| L4：特定条件内完全自動 | L3/L4：自律施工 | 環境定義が困難（地形が常に変化） |

### 共通技術

- **センサフュージョン**：LiDAR・カメラ・GNSS・IMUの統合
- **地図・自己位置推定**：RTK-GNSSによる高精度測位、LiDAR-SLAMによる局所地図
- **経路・動作計画**：衝突回避・最適経路探索（A\*・RRT等）
- **障害物認識**：物体検出（人・車両・障害物）

### 建設機械固有の難しさ

```
自動車                          建設機械
─────────────────────────────────────────────
道路（不変・既知の形状）      施工対象が常に変化（掘れば地形が変わる）
2次元の移動制御               作業機（アーム・バケット）の3次元精密制御
整備された舗装路              不整地・急傾斜・泥濘・岩盤
標準化された交通参加者        現場作業員（V2Pの重要性が高い）
GPSが届く環境が多い           地下・構造物の影・電波遮蔽環境が多い
自律化の主目的は移動          施工品質（出来形・精度）が直接の目標
```

一方で**易しい点**もある：作業速度が低速（衝突余裕時間が長い）、作業エリアが施工計画で事前に定義可能、歩行者・一般車両との混在がない管理された現場が多い。

### 各レイヤの技術要素

**センサフュージョン層**
- RTK-GNSS（測位精度 ±2〜3cm 程度、要確定）によるマシンコントロール基盤
- IMUによる姿勢推定（傾斜地での座標補正）
- LiDAR点群によるリアルタイム3Dマッピング
- カメラ画像による人検知・物体分類

**認識・計画層**
- 施工前後のポイントクラウド差分比較による出来形計測
- BIM/CIMデータ（設計形状）との照合・差分指示
- 動作計画：バケット軌道生成・油圧系への指令値変換

**制御層（Classic AUTOSAR）**
- 油圧アクチュエータへの精密フィードバック制御（PID・モデル予測制御）
- 上位計画層からの指令を安全範囲内でのみ実行するガーディング
- フェールサイレント・フェールセーフの実現

## 4. AUTOSARとの関係

### レイヤ別のAUTOSAR適用

| 機能 | 適用技術 | 理由 |
|------|----------|------|
| 油圧制御・フェールセーフ | Classic AUTOSAR | リアルタイム性・機能安全認証（AgPL d/e）が必須 |
| センサフュージョン・経路計画 | Linux（ROS2等） | 計算量が大きい・ライブラリエコシステムが充実 |
| クラウド・V2V通信 | Linux | TCP/IP・MQTT・TLSが前提 |
| 将来の統合認識基盤 | Adaptive AUTOSAR（2028年以降） | ara::com・ara::execによるSOAへの移行 |

### 機能安全要求の増大

自律化レベルが上がるほどAgPL（Agricultural Performance Level、ISO 25119準拠）の要求が高まる。

- **L0/L1（ガイダンス・軸補助）**：人間が最終判断→AgPL c程度
- **L2（作業サイクル自動化）**：人が監視するが制御は機械→AgPL d相当
- **L3/L4（高度・完全自律）**：人命に直結する判断を機械が実施→AgPL e相当（要確定）

AgPL e相当の機能ではHARAの対象故障モードが増加し、Classic AUTOSARの安全機構（E2E保護・監視犬・冗長センサ）の設計要件が厳しくなる。

### 人検知SWCの設計例

```
人検知SWC（Linux側）
 ├─ カメラ画像処理（物体検出NN）
 └─ 検知結果をCAN経由でClassic AUTOSAR ECUへ通知

人検知受信・安全監視SWC（Classic AUTOSAR側）
 ├─ 受信タイムアウト監視（E2E）
 ├─ 検知信号True受信→作業機動作制限指令
 └─ 通信断→フェールセーフ（作業機停止）
```

## 5. 先行事例と技術成熟度

| 企業 | 取り組み | 到達レベル（概況） |
|------|----------|-------------------|
| コマツ | スマートコンストラクション・無人ダンプ（鉱山） | 鉱山向けL3/L4（特定管理環境）、建設向けL0〜L1が普及段階 |
| キャタピラー | Cat Command（テレオペ・半自律） | 鉱山向け自律運搬が実用化。建設機械は遠隔操作主体 |
| 日立建機 | ConSite・Solution Linkage | フリート監視・予知保全中心。自律施工は開発段階 |
| 大成建設・清水建設等 | 国交省i-Construction実証 | ICT施工（L0ガイダンス）が実用化。協調施工実証中 |

**技術成熟度の整理**
- L0（ガイダンス）：**商用製品として普及済み**
- L1（軸補助）：**主要メーカーで標準装備化が進行中**
- L2（作業サイクル自動化）：**鉱山・特定環境で実用化、一般建設は実証段階**
- L3/L4（高度・完全自律）：**研究・実証段階**（一般建設現場では未実用）

## 6. 建設現場の安全要件

### 作業員との混在と規格対応

建設機械の自律化では、ISO 13849（機械安全・制御系の安全関連部）の適用が前提となる。ISO 25119はISO 13849を農業機械に適用した規格体系であり、建設機械も同様の考え方を採用する。

- **PL（Performance Level）の決定**：人検知失敗→作業員との衝突という故障経路はPL d〜e相当
- **チャンネル冗長性**：人検知はカメラ＋LiDAR等の異種センサ冗長が望ましい

### ジオフェンシング（立入禁止区域の電子管理）

- 施工エリア境界をGNSS座標で定義し、機械の侵入・逸脱を検知
- 作業員のウェアラブルタグとV2P通信による位置把握
- Classic AUTOSAR ECUでジオフェンス違反を監視→作業機・走行の自動停止

### 緊急停止

- ハードウェアインターロック（ソフトウェアを介さない物理回路）の必須化
- 遠隔操作時の通信断→フェールセーフ（その場停止）の設計原則
- UDSのECU Reset・緊急停止DTCの記録による事後解析

## 7. 自社への示唆・移行ロードマップ

### AUTOSARの機能安全基盤が自律化の前提になる理由

自律化機能の安全性は「認識・判断の正確さ」だけでなく、**制御層のフェールセーフ設計**に依存する。Classic AUTOSARのOS・Wdg・E2E・Dem等の安全機構が正しく実装されていなければ、上位の認識ソフトウェアがどれほど優秀でも安全な自律化は実現できない。

→ **AUTOSARへの移行完了が自律化投資の前提条件**

### 2026〜2035年 自律化ロードマップ（案）

```
2026〜2028年：AUTOSARへの移行完了・機能安全基盤整備
  ↓
  施工ガイダンス（L0）の品質向上・RTK-GNSS統合
  人検知SWCの設計・ISO 25119 HARAへの組み込み

2028〜2031年：半自律化（L1）の実装
  ↓
  バケット高さ・角度補助の自動制御
  LiDAR統合・3Dマッピングの基盤構築
  V2P通信（UWBウェアラブル等）の試験導入

2031〜2035年：作業サイクル自動化（L2）へ
  ↓
  Adaptive AUTOSAR検討（Hypervisor統合含む）
  BIM/CIM連携による自律掘削・整地サイクル
  複数台協調施工（V2V）の実証
```

### 現実的な出発点

現時点で最も投資対効果が高いのは**L0：施工ガイダンスの高度化**である。RTK-GNSSと設計データ（BIM/CIM）を活用した出来形リアルタイム表示は、AUTOSAR移行と並行して進められる。人検知SWCの設計をAUTOSAR SWC分割の早期から組み込むことで、将来の自律化拡張に備えた安全アーキテクチャを確立できる。
