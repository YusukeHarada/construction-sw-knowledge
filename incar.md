# In-car技術概念 — 建設機械SDVアーキテクチャにおける車内システム

---

## 1. In-carとは何か

**In-car**とは、車両・機械の**内部**に閉じたシステム・通信・UXの総称である。対義語は**Out-car**（クラウド連携・V2X・外部インフラとの通信）。

| 区分 | 対象範囲 |
|---|---|
| In-car | コックピット、HMI、車内ネットワーク（CAN/LIN/Ethernet）、HVAC、乗員安全、ECU間通信 |
| Out-car | クラウド通信、OTA、フリート管理、V2X、現場インフラ連携 |

建設機械におけるIn-car構成要素：

```
┌──────────────────────────────────────────────┐
│              建設機械キャブ内                 │
│  ┌──────────┐  ┌──────────┐  ┌───────────┐  │
│  │ デジタル │  │ アーム   │  │ カメラ   │  │
│  │ クラスタ │  │ 操作UI   │  │ ディスプレ│  │
│  └──────────┘  └──────────┘  │ イ（後方）│  │
│  ┌──────────────────────────┐  └───────────┘  │
│  │ HMI / タッチパネル      │                 │
│  └──────────────────────────┘                 │
│  ┌───────────┐  ┌───────────┐                 │
│  │ HVAC制御  │  │ 音声入力  │                 │
│  └───────────┘  └───────────┘                 │
│         ↕ 車内ネットワーク（CAN FD / Ethernet）  │
│  ┌────────┐ ┌────────┐ ┌────────┐             │
│  │制御ECU │ │ゲートウェ│ │ドライバ│             │
│  │（油圧）│ │ イECU   │ │モニタ  │             │
│  └────────┘ └────────┘ └────────┘             │
└──────────────────────────────────────────────┘
```

## 2. コックピット・HMI

### HMIの進化

```
アナログ計器盤 → デジタルクラスタ → タッチパネル統合 → 統合コックピット(ICC)
（現状）         （移行期）          （Phase 2）         （2028年〜）
```

### 建設機械向けIVI（In-Vehicle Infotainment）相当機能

| 機能 | 内容 |
|---|---|
| 作業支援情報表示 | 土量・積載重量・傾斜角・燃費リアルタイム表示 |
| 施工ガイダンス | 3Dマシンコントロール（ICT施工）との連携表示 |
| カメラ統合 | 後方・側方カメラ映像のOSD（On-Screen Display）合成 |
| アラーム管理 | 油圧異常・温度超過・DTC表示（UDS/J1939連携） |

### HMIプラットフォーム選択肢

| プラットフォーム | 特徴 | 建設機械適用性 |
|---|---|---|
| Android Automotive OS | アプリエコシステム豊富、Google Maps連携 | HMI・情報系に適用可（要確定） |
| Linux（Yocto） | カスタマイズ自由度高、OSS | 現実的な第一選択 |
| QNX | リアルタイム性・安全認証実績あり | 安全クリティカルUI（メータ等）向け |

### Classic AUTOSAR ECUとのデータ連携

HMIはLinux上で動作し、Classic AUTOSAR制御ECUからデータを受け取る構成：

```
[Classic AUTOSAR ECU]
  └─ SWC（信号送信）
      └─ COM / PduR
          └─ CAN FD / SOME/IP (SoAD経由)
              └─ [Linux HMI ECU]
                  └─ SOME/IP クライアント（vsomeip等）
                      └─ HMIアプリケーション（Qt / Flutter / Android）
```

## 3. 車内ネットワーク

### ECU間通信プロトコル比較

| プロトコル | 帯域 | 主な用途 | 建設機械での位置づけ |
|---|---|---|---|
| CAN FD | 最大8 Mbps | 制御系・診断 | 現在のメイン通信路 |
| LIN | 20 kbps | HVAC・シート・窓 | 低速アクチュエータ制御 |
| Ethernet（100BASE-T1等） | 100 Mbps〜 | カメラ・HMI・ゲートウェイ | 段階導入（要確定） |
| FlexRay | 10 Mbps | 高信頼制御（自動車） | 建設機械では採用例少 |

### ドメイン内・ドメイン間通信

```
[制御ドメイン]          [情報ドメイン]
 油圧ECU                HMI ECU
 走行ECU   ← CAN FD → ゲートウェイECU ← Ethernet → カメラECU
 PTO ECU                                              音声UI ECU
```

**車載Ethernetスイッチ（TSNスイッチ）の役割：**
- 複数ECU間の帯域保証（IEEE 802.1Qbv スケジューリング）
- 優先度制御による制御系遅延保証
- セキュリティ境界（VLAN分離）

### セキュリティ（In-car内部脅威対応）

In-carはOut-carより信頼度が高い領域だが、内部脅威（診断ポート経由・サプライヤECU改ざん等）への対策は必要：

- SecOC（Secure Onboard Communication）：CANメッセージへのMAC付与
- VLAN分離：制御系と情報系の通信分離
- 診断ポート認証：UDS 0x27（Security Access）によるアクセス制御

## 4. 統合コックピットコントローラ（ICC: Integrated Cockpit Controller）

### ICCの概念

従来は個別ECUだったHMI・デジタルクラスタ・HUD・音声UIを**1つの高性能SoC**に統合するトレンド。

```
従来構成：
 [クラスタECU] [HMI ECU] [音声UI ECU]  ← 3ECU

ICC構成：
 [ICC SoC（例：Qualcomm SA8155等・要確定）]
  ├─ VM1: 安全UI（ISO 25119 対応・速度/警告表示）← QNX等
  └─ VM2: 情報UI（施工ガイダンス・カメラ・音声）← Android/Linux
```

### 建設機械への適用

- 2028年以降の次世代機で検討（現状はスキル・認証実績の観点からデュアルSoC物理分離が現実解）
- Hypervisorによる安全UI（メータ・警告）と情報UI（HMI・カメラ）の分離が前提

## 5. 乗員安全・快適性

### ドライバモニタリングシステム（DMS）

建設機械オペレータの長時間作業における疲労・脇見検知：

| 検知項目 | センサ | 活用 |
|---|---|---|
| 眠気・疲労 | 赤外線カメラ（目の開度） | アラーム・作業停止提案 |
| 脇見 | カメラ＋視線推定AI | 危険操作インターロック連携 |
| 体調異変 | 心拍センサ（シート埋込・要確定） | 緊急停止連携 |

### HVAC・シート制御

- HVACコントローラ：LIN経由でモータ・バルブ制御
- エアサスシート：LIN/CAN経由で振動吸収制御（長時間作業の腰部負荷軽減）

## 6. 建設機械固有の考慮事項

| 課題 | 内容 | 対策 |
|---|---|---|
| 手袋操作 | タッチパネルが反応しない | 静電容量式（手袋対応）または物理ボタン併用 |
| 振動環境 | キャブ内の激しい振動 | 表示固定・タッチ誤操作防止ロジック |
| 防塵・防水 | 粉塵・泥水の侵入 | ディスプレイ・ECUのIP67相当対応（要確定） |
| 可読性 | 直射日光下の視認性 | 高輝度ディスプレイ（1000 nit以上・要確定） |
| 建機固有UI | アーム角度・積載荷重・傾斜表示 | 専用ウィジェット・グラフィカル表示 |

## 7. In-car移行ロードマップ

```
現状（〜2025年）
  アナログ計器・独立HMI・CAN主体・カメラ単体表示
    ↓
Phase 1（2025〜2027年）
  統合ディスプレイ導入・SOME/IP連携・カメラOSD統合
  Linux（Yocto）HMI基盤構築・CAN FD移行
    ↓
Phase 2（2027〜2029年）
  統合コックピット（HMI＋クラスタ＋カメラ統合）
  Android Automotive OS または Linux HMI プラットフォーム確立
  DMS（ドライバモニタリング）導入
    ↓
Phase 3（2029年〜）
  ICC（統合コックピットコントローラ）＋Hypervisor
  HUD（ヘッドアップディスプレイ）
  音声AI連携（自然言語による機械操作・作業指示）
```
